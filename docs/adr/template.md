## Context（背景）

把當時面對的選擇與限制寫清楚。1~3 段。避免「我們很棒所以選 X」這種無內容話。

寫進來的應該是：
- 觸發這個決定的具體情境（哪個 milestone、什麼 trigger）
- 已知的限制（工時、技術、法律、平台）
- 不寫進來會被未來自己誤解的前提

## Decision（決定）

明確的決定 + 一句話總結 why。例：

> 採用 X 而非 Y，因為在 <限制 1> 與 <限制 2> 下，X 的 <優點> 直接命中 v1 的 <DoD 條件>。

## Consequences（後果）

- **正面後果**（具體，不寫「會比較好」這種空話）
- **負面後果**（誠實寫出 — ADR 不是行銷文案）
- **開放問題 / 風險 / 緩解**（已知但暫不解的問題；什麼條件會觸發重評）

## Alternatives considered（替代方案）

至少列 2 個沒選的方案，每個寫一句「為什麼沒選」。沒考慮過就直說沒考慮過。

- **方案 A**：說明 + 為什麼沒選
- **方案 B**：說明 + 為什麼沒選
